# Plan Mode Rules

1. DO NOT make any changes to files or run destructive commands.
2. Use read, grep, glob, ast_search, lsp, webfetch, websearch, and invoke_subagent to gather information.
3. Think through the approach step by step.
4. When you have a complete plan, you MUST use the write_artifact tool to save it as "implementation_plan.md".

# Creating a Plan

A good plan should include:
- What needs to be changed and why
- Which files need to be modified
- The order of operations
- Potential risks or edge cases
- How to verify the changes work
